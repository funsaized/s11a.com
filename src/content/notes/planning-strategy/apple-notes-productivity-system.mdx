TAG SYSTEM

Status Tags (Workflow)

- Needs processing/filing

- Currently working on

- Blocked or pending response

- Needs regular check-in

- Future consideration

- Completed (before archiving)

Context Tags (Domain)

- Development work

- Technical research/learning

- Streaming/writing/social

- Project/strategic planning

- Non-work related

- Meeting notes/agendas

Priority Tags

- Immediate attention needed

- High impact, schedule soon

- Tasks under 15 minutes

Project Tags (Dynamic)

- Specific project identifier

- Client-specific work

- Platform-specific (iOS, web, etc.)

SCHEDULED WORKFLOW SYSTEM

Daily Workflows

Morning Startup (5 minutes)

Time: First thing when opening laptop

Actions:

1\. Open Inbox folder

1\. Process all  tagged notes:

• File to appropriate folder

• Add context/priority tags

• Set status tag

1\. Review  tagged notes

1\. Check  items for today’s focus

Evening Wrap-up (3 minutes)

Time: End of workday

Actions:

1\. Capture any loose thoughts to Inbox

1\. Update status on  items

1\. Tag tomorrow’s priorities with  or

Weekly Workflows

Monday Planning (10 minutes)

Time: Monday morning after daily startup

Actions:

1\. Review  projects

1\. Check  items for follow-up

1\. Move completed items to Archive

1\. Plan week’s content creation priorities

1\. Update project roadmaps in Planning folder

Friday Review (15 minutes)

Time: Friday afternoon

Actions:

1\. Archive completed work (#done → Archive folder)

1\. Review  items for potential activation

1\. Clean up duplicate or outdated notes

1\. Update project status in Active Projects

1\. Plan next week’s priorities

Monthly Workflows

Monthly Cleanup (30 minutes)

Time: Last Friday of month

Actions:

1\. Archive Sweep: Move old completed items to Archive

1\. Tag Audit: Remove unused project tags, consolidate similar tags

1\. Folder Review: Ensure notes are in correct folders

1\. Template Update: Refresh note templates based on evolved needs

1\. Knowledge Base: Consolidate scattered research into reference docs

1\. System Review: Assess what’s working/not working in the system

MAINTENANCE RULES

The Two-Minute Rule

If processing a note takes less than 2 minutes, do it immediately rather than leaving it in Inbox.

Weekly Inbox Zero

Inbox should be empty every Friday. If not, schedule time to process remaining items.

Tag Hygiene

• Maximum 5 tags per note

• Remove tags when they’re no longer relevant

• Create new project tags sparingly

Archive Criteria

Move to Archive when:

• Project is completed

• Information is over 6 months old

• Note hasn’t been accessed in 3+ months

• Reference material is superseded
